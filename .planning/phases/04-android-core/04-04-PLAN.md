**Part A -- Update OutlinerGodTheme to support density:**

Modify `OutlinerGodTheme.kt` to accept a density parameter and apply it via `CompositionLocalProvider`:

```kotlin
package com.gmaingret.outlinergod.ui.theme

import androidx.compose.foundation.isSystemInDarkTheme
import androidx.compose.material3.MaterialTheme
import androidx.compose.material3.Typography
import androidx.compose.material3.darkColorScheme
import androidx.compose.material3.lightColorScheme
import androidx.compose.runtime.Composable
import androidx.compose.runtime.CompositionLocalProvider
import androidx.compose.ui.platform.LocalDensity
import androidx.compose.ui.unit.Density

@Composable
fun OutlinerGodTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    densityScale: Float = 1.0f,
    content: @Composable () -> Unit
) {
    val customDensity = Density(
        density = LocalDensity.current.density * densityScale,
        fontScale = LocalDensity.current.fontScale
    )

    MaterialTheme(
        colorScheme = if (darkTheme) darkColorScheme() else lightColorScheme(),
        typography = Typography(),
    ) {
        CompositionLocalProvider(
            LocalDensity provides customDensity,
            content = content,
        )
    }
}
```

The `densityScale` parameter controls UI density:
- "cozy" = 1.0f (default, normal spacing)
- "comfortable" = 0.95f (slightly tighter)
- "compact" = 0.85f (noticeably tighter spacing)

These values are mapped in the caller (MainActivity), not in the theme itself.

**Part B -- Update MainActivity to observe settings:**

MainActivity needs to:
1. Inject `SettingsDao` and `AuthRepository` (or a simpler mechanism to get userId)
2. Observe `settingsDao.getSettings(userId)` as a Compose state
3. Map settings to theme parameters

Since MainActivity is an `@AndroidEntryPoint`, it can inject dependencies. However, observing settings requires knowing the userId. The cleanest approach: use `flatMapLatest` on the access token Flow to switch to the correct user's settings.

Modify `MainActivity.kt`:

```kotlin
package com.gmaingret.outlinergod

import android.os.Bundle
import androidx.activity.ComponentActivity
import androidx.activity.compose.setContent
import androidx.activity.enableEdgeToEdge
import androidx.compose.runtime.collectAsState
import androidx.compose.runtime.getValue
import androidx.navigation.compose.rememberNavController
import com.gmaingret.outlinergod.db.dao.SettingsDao
import com.gmaingret.outlinergod.repository.AuthRepository
import com.gmaingret.outlinergod.ui.navigation.AppNavHost
import com.gmaingret.outlinergod.ui.theme.OutlinerGodTheme
import dagger.hilt.android.AndroidEntryPoint
import kotlinx.coroutines.flow.flatMapLatest
import kotlinx.coroutines.flow.flowOf
import javax.inject.Inject

@AndroidEntryPoint
class MainActivity : ComponentActivity() {

    @Inject lateinit var settingsDao: SettingsDao
    @Inject lateinit var authRepository: AuthRepository

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()

        // Create a Flow that observes the current user's settings.
        // If no user is logged in (token is null), emit null settings.
        val settingsFlow = authRepository.getAccessToken()
            .flatMapLatest { accessToken ->
                if (accessToken != null) {
                    settingsDao.getSettings(accessToken)
                } else {
                    flowOf(null)
                }
            }

        setContent {
            val settings by settingsFlow.collectAsState(initial = null)

            val isDarkTheme = when (settings?.theme) {
                "light" -> false
                "dark" -> true
                else -> true // default to dark if no settings yet
            }

            val densityScale = when (settings?.density) {
                "cozy" -> 1.0f
                "comfortable" -> 0.95f
                "compact" -> 0.85f
                else -> 1.0f
            }

            OutlinerGodTheme(
                darkTheme = isDarkTheme,
                densityScale = densityScale,
            ) {
                val navController = rememberNavController()
                AppNavHost(navController = navController)
            }
        }
    }
}
```

Key design decisions:
- `flatMapLatest` on `getAccessToken()` ensures settings update when user changes (login/logout).
- Default to dark theme when settings are null (before login or during loading) -- this matches the SettingsEntity default of `theme = "dark"`.
- `SettingsDao` is injected directly into the Activity via `@Inject lateinit var`. This is valid because `@AndroidEntryPoint` supports field injection.
- The density mapping uses scale factors (0.85-1.0) that provide noticeable but not extreme density changes.

**Part C -- Add @OptIn for flatMapLatest if needed:**

`flatMapLatest` is in `kotlinx.coroutines.flow` and requires `@OptIn(ExperimentalCoroutinesApi::class)` in some Kotlin/coroutines versions. Add the annotation to `onCreate()` or to the class if the compiler requires it. Check during build -- if it compiles without, skip the annotation.

**What NOT to do:**
- Do NOT create a new ViewModel for MainActivity. Direct DAO observation in the Activity's setContent is fine for a single Flow that controls the theme wrapper.
- Do NOT change SettingsViewModel or SettingsScreen -- they already work correctly.
- Do NOT add a "system" theme option. Per the UAT and SettingsScreen UI, the options are "dark" and "light" only.
