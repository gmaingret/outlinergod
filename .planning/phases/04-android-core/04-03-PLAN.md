The problem: `onValueChange` in BasicTextField only fires when text actually changes. Pressing backspace in an empty field has nothing to delete, so Android never calls `onValueChange`. The condition `if (oldText.isEmpty() && newText.isEmpty())` on line 317 is dead code.

The fix: Use a zero-width space (ZWS, `\u200B`) as a sentinel character. When the node content is empty, the `TextFieldValue` holds `"\u200B"` instead of `""`. When the user presses backspace, the ZWS is deleted, triggering `onValueChange` with newText="" while oldText="\u200B". We detect this and fire `onBackspaceOnEmpty()`.

**Step 1 -- Define a sentinel constant at file level (or inside NodeRow):**
Add near the top of the file (after imports, before the composable functions):
```kotlin
/** Zero-width space used as a sentinel so backspace fires onValueChange even on empty nodes. */
private const val ZWS = "\u200B"
```

**Step 2 -- Update the textFieldValue initialization in NodeRow (line 236-238):**
Change:
```kotlin
var textFieldValue by remember(flatNode.entity.id) {
    mutableStateOf(TextFieldValue(flatNode.entity.content))
}
```
To:
```kotlin
var textFieldValue by remember(flatNode.entity.id) {
    val displayText = flatNode.entity.content.ifEmpty { ZWS }
    mutableStateOf(TextFieldValue(displayText, selection = TextRange(displayText.length)))
}
```
When content is empty, the field starts with the ZWS sentinel. Cursor is placed after the ZWS so the user's backspace deletes it.

**Step 3 -- Update the onValueChange handler (lines 304-324):**
Replace the entire `onValueChange` lambda with:
```kotlin
onValueChange = { newValue ->
    val newText = newValue.text
    val oldText = textFieldValue.text

    // Detect Enter key: newline inserted
    val newlineIndex = newText.indexOf('\n')
    if (newlineIndex >= 0 && !oldText.contains('\n')) {
        // Strip any ZWS before computing cursor position
        val cleanPosition = newText.substring(0, newlineIndex).replace(ZWS, "").length
        onEnterPressed(cleanPosition)
        return@BasicTextField
    }

    // Detect Backspace on empty: ZWS was deleted, leaving empty string
    if (oldText == ZWS && newText.isEmpty()) {
        onBackspaceOnEmpty()
        return@BasicTextField
    }

    // Normal text change: strip ZWS from the text if user typed real characters alongside it
    val cleanText = newText.replace(ZWS, "")
    if (cleanText.isEmpty()) {
        // User deleted all real content -- restore sentinel
        textFieldValue = TextFieldValue(ZWS, selection = TextRange(1))
        onContentChanged("")
    } else {
        // Remove any residual ZWS and update normally
        if (newText.contains(ZWS)) {
            val cursorOffset = newValue.selection.start - (newText.length - cleanText.length)
            textFieldValue = TextFieldValue(cleanText, selection = TextRange(cursorOffset.coerceAtLeast(0)))
        } else {
            textFieldValue = newValue
        }
        onContentChanged(cleanText)
    }
},
```

Key behaviors:
- Backspace on empty: `oldText == ZWS && newText.isEmpty()` fires `onBackspaceOnEmpty()`
- Enter with ZWS present: strip ZWS before computing cursor position for content split
- User types real characters: ZWS is stripped so it never appears in saved content
- User deletes all real characters: ZWS is restored as sentinel

**Step 4 -- Update the VisualTransformation:**
The ZWS character is invisible by default, so `MarkdownVisualTransformation` does not need changes. However, if there are any cursor offset issues, the ZWS is automatically stripped during normal editing. Verify that MarkdownVisualTransformation does not count/transform ZWS characters specially.

**What NOT to do:**
- Do NOT use `KeyEvent` interception or `InputTransformation` (those require TextFieldState API, but this codebase uses old-style BasicTextField with `value`/`onValueChange`).
- Do NOT change NodeEditorViewModel. The `onBackspaceOnEmptyNode()` ViewModel method works correctly -- the problem is purely that the UI never calls it.
