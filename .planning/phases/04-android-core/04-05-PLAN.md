<task type="auto">
  <name>Task 2: Move long-press detection from Column ancestor into BasicTextField modifier</name>
  <files>android/app/src/main/java/com/gmaingret/outlinergod/ui/screen/nodeeditor/NodeEditorScreen.kt</files>
  <action>
In NodeRow, the content area is currently:

  Column(
      modifier = Modifier
          .weight(1f)
          .combinedClickable(
              onClick = { /* regular tap handled by text field focus */ },
              onLongClick = onLongPress,
          )
  ) {
      BasicTextField(...)
  }

BasicTextField installs its own pointer handler internally, which consumes all pointer events on pointer-down before they bubble up to the wrapping Column. combinedClickable on any ancestor of BasicTextField therefore never fires.

Make these two changes:

1. Remove combinedClickable from the wrapping Column entirely. Replace with plain Modifier.weight(1f):

  Column(
      modifier = Modifier.weight(1f)
  ) {
      BasicTextField(...)
  }

2. Add a pointerInput block to the BasicTextField's modifier chain, AFTER fillMaxWidth() and BEFORE focusRequester():

  modifier = Modifier
      .fillMaxWidth()
      .pointerInput(Unit) {
          detectTapGestures(onLongPress = { onLongPress() })
      }
      .focusRequester(focusRequester)
      .onFocusChanged { ... }

Add the missing import at the top of the file:
  import androidx.compose.foundation.gestures.detectTapGestures

Why: pointerInput placed directly on a BasicTextField modifier is processed inside the same gesture pipeline as BasicTextField's own handlers. detectTapGestures with onLongPress fires on a sustained press regardless of which internal handler claims the tap — the long-press timer starts on pointer-down and fires if the pointer has not moved and not been released after the long-press timeout (~500ms). This placement sidesteps the event-consumption problem that ancestor-level combinedClickable has with BasicTextField.
