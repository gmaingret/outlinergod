**Step A — Rewrite `backend/Dockerfile`**

Replace the entire contents of `backend/Dockerfile` with the three-stage build below. The context is now the project root, so all `COPY` paths must be prefixed with `backend/` or `web/` as appropriate.

```dockerfile
# Stage 1: Build React SPA
FROM node:20-alpine AS web-builder
RUN corepack enable && corepack prepare pnpm@latest --activate
WORKDIR /web
COPY web/package.json web/pnpm-lock.yaml ./
RUN pnpm install --frozen-lockfile
COPY web/ ./
RUN pnpm run build

# Stage 2: Compile backend TypeScript
FROM node:20-alpine AS ts-builder
RUN corepack enable && corepack prepare pnpm@latest --activate
WORKDIR /app
RUN apk add --no-cache python3 make g++
COPY backend/package.json backend/pnpm-lock.yaml ./
RUN pnpm install --frozen-lockfile
COPY backend/tsconfig.json ./
COPY backend/src/ ./src/
RUN pnpm run build

# Stage 3: Lean production runner
FROM node:20-alpine AS runner
WORKDIR /app
COPY --from=ts-builder /app/dist ./dist
COPY --from=ts-builder /app/node_modules ./node_modules
COPY --from=ts-builder /app/package.json ./
COPY backend/drizzle ./drizzle
COPY --from=web-builder /web/dist ./web/dist
RUN mkdir -p /data/uploads && chown -R node:node /data
EXPOSE 3000
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD node -e "require('http').get('http://localhost:3000/health',(r)=>process.exit(r.statusCode===200?0:1))"
USER node
CMD ["node", "dist/server.js"]
```

Key changes from the original:
- Added Stage 1 (`web-builder`) that builds the React SPA from `web/`
- Stage 2 (`ts-builder`) is the renamed original `builder` stage — all `COPY` paths now have `backend/` prefix because context moved from `./backend` to `.`
- Stage 3 (`runner`) adds `COPY --from=web-builder /web/dist ./web/dist` — this places the React bundle at `/app/web/dist` inside the container, which matches the `WEB_DIST_PATH` the backend resolves at startup

**Step B — Update `docker-compose.yml`**

Replace the entire `docker-compose.yml` with:
```yaml

    env_file: .env
    restart: unless-stopped

volumes:
  outlinergod-data:
```

The only changes are `context: .` (was `./backend`) and `dockerfile: backend/Dockerfile` (explicit path since context is no longer the Dockerfile's directory).

**Step C — Create root `.dockerignore`**

Create `.dockerignore` at the project root with:

Note: `web/dist/` is excluded from the build context because the web-builder stage compiles from source. Including `web/dist/` would waste context size without benefit.

The existing `backend/.dockerignore` continues to exist but has no effect (Docker reads `.dockerignore` from the context root, which is now the project root).
