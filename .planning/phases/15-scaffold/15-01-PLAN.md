Changes to `buildApp()`:

1. Add imports at the top of the file (after existing imports):
```typescript
import fastifyStatic from '@fastify/static'
import { fileURLToPath } from 'node:url'
import { join } from 'node:path'
```

2. Remove the placeholder route at the bottom of `buildApp()`:
```typescript
// DELETE THIS — replace with @fastify/static registration below:
app.get('/', async (_req, reply) => {
  return reply.send({ message: 'OutlinerGod API' })
})
```

3. After all `app.register(create*Routes...)` calls, add the static serving block:
```typescript
// Serve React SPA from web/dist.
// In production Docker: /app/web/dist (set WEB_DIST_PATH env var).
// In dev (tsx src/server.ts): computed relative to src/ → ../../web/dist.
const __dirname = fileURLToPath(new URL('.', import.meta.url))
const webDistPath = process.env.WEB_DIST_PATH ?? join(__dirname, '..', '..', 'web', 'dist')

void app.register(fastifyStatic, {
  root: webDistPath,
  wildcard: false,  // CRITICAL: do not use wildcard: true — it creates GET /* that shadows /api routes
})

// SPA fallback: any GET that doesn't match an API route or static file returns index.html.
// This makes deep links work (e.g. /documents/123 loads the app, React Router handles routing).
app.setNotFoundHandler((_req, reply) => {
  return reply.sendFile('index.html')
})
```

ORDERING: The `fastifyStatic` registration and `setNotFoundHandler` MUST come AFTER all `app.register(create*Routes...)` calls. The existing route registrations in the file are already in the right place — just replace the placeholder `GET /` at the bottom.

CRITICAL: `wildcard: false` is mandatory. Using the default `wildcard: true` registers a `GET /*` route that intercepts `/api/*` calls and returns static file 404s instead of JSON. See STATE.md CRITICAL PITFALLS.

The `web/dist/` directory will not exist in development until `pnpm build` is run in `web/`. Fastify will log an error at startup if the path does not exist, but this is acceptable for the dev workflow (developer runs `pnpm build` first or uses `pnpm dev` in the web/ directory for HMR).

After modifying `index.ts`, run the full backend test suite to confirm no regression on /api/* routes:
```bash
cd backend && pnpm test
```
