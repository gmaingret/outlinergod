    **flattenTree()** — DFS traversal respecting `collapsed` state:
    ```typescript
    export function flattenTree(
      roots: TreeNode[],
      depth = 0,
      parentId: string | null = null,
      result: FlatNode[] = []
    ): FlatNode[] {
      for (const node of roots) {
        result.push({ node, depth, parentId })
        if (!node.collapsed && node.children.length > 0) {
          flattenTree(node.children, depth + 1, node.id, result)
        }
      }
      return result
    }
    ```

    **reorderNode()** — algorithm:
    1. Call `flattenTree(roots)` to get current visible flat list
    2. Find dragged node's entry in flat list (`activeIndex`)
    3. Collect the dragged node's full subtree (including collapsed children) by DFS on the TreeNode — this is the "block" to move
    4. Determine `newParentId`:
       - If `newDepth === 0`: `newParentId = null`
       - Else: look backwards from `targetIndex - 1` in flatNodes to find the first FlatNode at `newDepth - 1`; its `node.id` is the new parent. Use the `parentId` chain on FlatNode for ancestor resolution.
    5. Remove dragged node from its current parent's children (or roots if no parent)
    6. Call `recomputeSortOrders` on the old sibling list
    7. Find the new parent node in the tree and insert the dragged node at the correct position:
       - Position within new parent's children is derived from `targetIndex` relative to how many of the new parent's existing children appear before `targetIndex` in the flat list
       - Simplest correct approach: append to new parent's children list, then call `recomputeSortOrders` on new sibling list (positions will be correct because the flat list insertion index determines relative order)
    8. Update `node.parent_id = newParentId` on the dragged node
    9. Return the mutated roots

    **Clamping rule** (implement in reorderNode, not getProjection):
    - `newDepth` passed in is already clamped by the caller (FlatNodeList); reorderNode trusts its inputs
    - If no valid parent found at `newDepth - 1`, fall back to depth 0 (root insertion)

    **Important**: `reorderNode` mutates the tree nodes in place (same pattern as `indentNode`/`outdentNode`). Return the same `roots` array reference with mutations applied.
