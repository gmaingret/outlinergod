buildTree:
```typescript
function buildTree(nodes: NodeResponse[]): TreeNode[] {
  const map = new Map<string, TreeNode>()
  const roots: TreeNode[] = []
  for (const n of nodes) map.set(n.id, { ...n, children: [] })
  for (const n of nodes) {
    const node = map.get(n.id)!
    if (n.parent_id && map.has(n.parent_id)) {
      map.get(n.parent_id)!.children.push(node)
    } else {
      roots.push(node)
    }
  }
  // Sort lexicographically by sort_order at every level
  const sortChildren = (nodes: TreeNode[]) => {
    nodes.sort((a, b) => a.sort_order < b.sort_order ? -1 : a.sort_order > b.sort_order ? 1 : 0)
    for (const n of nodes) sortChildren(n.children)
  }
  sortChildren(roots)
  return roots
}
```

filterSubtree (BFS):
```typescript
function filterSubtree(roots: TreeNode[], rootNodeId: string): TreeNode | null {
  const queue: TreeNode[] = [...roots]
  while (queue.length) {
    const node = queue.shift()!
    if (node.id === rootNodeId) return node
    queue.push(...node.children)
  }
  return null
}
```

recomputeSortOrders: assign sort_order = 'a' + index.toString().padStart(4, '0') for each child in a siblings array (e.g., 'a0000', 'a0001', 'a0002'). This matches Android recomputeSortOrders() — simple, monotonically increasing, lexicographically valid.

insertSiblingNode: find the target node's parent (or root list), insert a new TreeNode immediately after it, call recomputeSortOrders on the siblings, return the mutated root tree and the new node's id.

indentNode: find the target node and its preceding sibling in DFS order. If no preceding sibling (it is first child or root), do nothing. Otherwise: remove target from current parent's children, append to preceding sibling's children, recompute sort orders on both affected sibling lists.

outdentNode: find the target node and its parent. If no parent (root-level node), do nothing. Otherwise: remove target from parent's children, insert target immediately after parent in grandparent's children list (or root list), recompute sort orders on both affected lists.

CRITICAL: sort_order is TEXT — use lexicographic comparison only. 'aV' < 'b0' is correct. Never call parseFloat or Number() on sort_order.
