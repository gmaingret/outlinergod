## Issues Encountered

- **Room 2.8.4 + kotlinx.serialization 1.7.3 AbstractMethodError**: KSP's FieldBundle$$serializer (compiled against serialization 1.6.x, bundled in room-compiler JAR) throws AbstractMethodError for `typeParametersSerializers()` when KSP needs to regenerate schema output. Root cause: this method was added to GeneratedSerializer interface in serialization 1.7.0. The fix: keep 3.json committed so KSP stays UP-TO-DATE and never needs to re-run.
- **Android Studio parallel Gradle builds**: FileAlreadyExistsException in KSP output directory when Android Studio built simultaneously with CLI. Resolved by waiting for Android Studio to settle before running CLI build.
