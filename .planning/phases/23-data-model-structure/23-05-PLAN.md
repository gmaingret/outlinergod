<task type="auto">
  <name>Task 1: Remove destructive doFirst hook from build.gradle.kts</name>
  <files>android/app/build.gradle.kts</files>
  <action>
Delete the entire doFirst block (lines 161-172 in the current file) from android/app/build.gradle.kts. This block matches all ksp*Kotlin tasks and deletes the schemas/ directory before every KSP run.

The block to remove is:

```
// Workaround: Room 2.8.4's FieldBundle$$serializer (compiled without typeParametersSerializers()
// added to GeneratedSerializer in kotlinx-serialization 1.7.0) crashes when reading an existing
// schema JSON with serialization 1.7+ on the classpath.  Delete previous schema before KSP runs
// so Room only writes a fresh schema; it cannot crash reading a file that does not exist.
tasks.matching { it.name.startsWith("ksp") && it.name.endsWith("Kotlin") }.configureEach {
    doFirst {
        val schemasDir = file("$projectDir/schemas")
        if (schemasDir.exists()) {
            schemasDir.deleteRecursively()
        }
    }
}
```

Rationale per D-23-04-A: once 3.json is committed and entities are stable, KSP stays UP-TO-DATE and never reads the existing schema JSON — so the AbstractMethodError that motivated the workaround cannot fire. The workaround was only needed when 3.json existed AND KSP re-ran; removing it breaks the delete-then-crash cycle.

Do NOT modify any other section of build.gradle.kts — leave the ksp { arg("room.schemaLocation", ...) } block untouched.
