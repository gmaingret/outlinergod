# Phase 22: Android performance - Context

**Gathered:** 2026-03-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Eliminate four identified Android performance bottlenecks: per-keystroke full-document fetch, unbatched DnD updates, 62-sibling sort-order overflow, and SyncLogger always active in release builds. No new behaviour — pure fixes to existing code paths.

</domain>

<decisions>
## Implementation Decisions

### Sort-order overflow (recomputeFlatNodes)
- Replace the clamped `"a${DIGITS[charIdx]}"` assignment with `FractionalIndex.generateNKeysBetween(null, null, indices.size)` per sibling group
- Produces unique, lexicographically correct keys for any sibling count (no cap at 62)
- No DB migration — old `"aX"` keys remain until the user performs a drag that triggers recomputeFlatNodes; they remain valid and sortable in the meantime
- New test in `FlatNodeMapperTest` (pure unit test, no ViewModel overhead): verifies that n=100 siblings produce 100 distinct, ascending sort orders after recompute

### DnD batch write (persistReorderedNodes)
- Add `@Transaction @Update suspend fun updateNodes(nodes: List<NodeEntity>)` to `NodeDao` — Room batches all updates in a single SQLite transaction
- Filter unchanged nodes before calling: build a list of only `NodeEntity` instances where `sortOrder` or `parentId` changed (with updated HLC/timestamp fields); call `updateNodes(changed)` only if the list is non-empty
- No need to inject `AppDatabase` into the ViewModel — the DAO annotation handles the transaction boundary

### SyncLogger DEBUG guard
- Guard inside the object itself: add `if (!BuildConfig.DEBUG) return` as the first line of `log()` and `logError()`
- `getLogs()` also returns `""` early if `!BuildConfig.DEBUG` — consistent with the ring buffer never being populated in release
- No call-site changes needed

### getNodeByIdSync — new DAO method + scope
- Add `@Query("SELECT * FROM nodes WHERE id = :nodeId AND deleted_at IS NULL LIMIT 1") suspend fun getNodeByIdSync(nodeId: String): NodeEntity?` to `NodeDao`
- Fix **all 3** call sites in `NodeEditorViewModel`:
  - `persistContentChange` (line 690): replace `getNodesByDocumentSync(documentId).firstOrNull { it.id == nodeId }`
  - `persistNoteChange` (line 706): same replacement
  - `onNodeFocusLost` (line 663): same replacement — single-node fetch is correct even for the diff-check path

### Claude's Discretion
- Exact HLC/timestamp field assignments inside `persistReorderedNodes` change list (same logic as current, just restructured)
- Whether `FlatNodeMapper.recomputeFlatNodes` is extracted into a helper or kept as a companion object function
- Test helper for creating mock `FlatNode` / `NodeEntity` instances in `FlatNodeMapperTest`

</decisions>

- `@Query` + `suspend fun` for sync DAO reads — matches `getNodesByDocumentSync`, `getPendingChanges`, etc.

- `if (!BuildConfig.DEBUG) return` guard — standard Android pattern; `BuildConfig` is auto-generated by AGP
