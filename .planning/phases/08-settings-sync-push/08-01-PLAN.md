**1. Add `id` field to `SettingsSyncRecord` in `network/model/Sync.kt` (line 76):**

Add `@SerialName("id") val id: String = ""` as the FIRST field in `SettingsSyncRecord`, before `userId`. This is required because the backend `upsertSettings()` prepared statement binds `@id` as the SQLite primary key. Without it, Android pushes would cause a NOT NULL violation on the backend. The default `""` is safe because the pull direction uses `ignoreUnknownKeys = true` and existing call sites construct `SettingsSyncRecord` only via `toSettingsEntity()` (which ignores `id`). The push direction will set `id = userId` via the mapper.

The updated data class should be:
```kotlin
@Serializable
data class SettingsSyncRecord(
    @SerialName("id") val id: String = "",
    @SerialName("user_id") val userId: String,
    // ... rest unchanged
)
```

**2. Add `getPendingSettings()` to `SettingsDao` in `db/dao/SettingsDao.kt`:**

Add this method to the existing `SettingsDao` interface:

@Query("""

    WHERE user_id = :userId

This is a pure `@Query` — no schema change, no migration, no version bump needed.
