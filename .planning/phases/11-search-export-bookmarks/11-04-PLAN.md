**2. Create `ui/screen/search/SearchViewModel.kt`:**
- `@HiltViewModel class SearchViewModel @Inject constructor(searchRepository: SearchRepository, authRepository: AuthRepository, documentDao: DocumentDao)`
- Extends `ViewModel(), ContainerHost<SearchUiState, SearchSideEffect>`
- `sealed class SearchUiState`:
  - `Idle` (no query entered)
  - `Loading`
  - `Success(results: List<SearchResultItem>, query: String)` where `SearchResultItem` is a data class: `data class SearchResultItem(val nodeId: String, val documentId: String, val content: String, val note: String, val documentTitle: String)`
  - `NoResults(query: String)`
  - `Error(message: String)`
- `sealed class SearchSideEffect`:
  - `NavigateToNodeEditor(documentId: String)`
- Private `val queryFlow = MutableStateFlow("")`
- In `init { intent {} }`:
  - Collect `queryFlow.debounce(400L).distinctUntilChanged().flatMapLatest { q -> ... }`:
    - If `q.isBlank()`: emit `SearchUiState.Idle`
    - Else: emit `Loading`, then try to call `searchRepository.searchNodes(q, userId)`. For each NodeEntity result, look up the document title via `documentDao.getDocumentByIdSync(node.documentId)?.title ?: "Unknown"`. Map to `SearchResultItem`. If results empty, emit `NoResults(q)`. If results non-empty, emit `Success(results, q)`. On exception, emit `Error(message)`.
  - Collect the flow and reduce each emission into state.
- `fun onQueryChanged(query: String)`: set `queryFlow.value = query`
- `fun onResultTapped(item: SearchResultItem) = intent {}`: `postSideEffect(NavigateToNodeEditor(item.documentId))`

**3. Create `ui/screen/search/SearchScreen.kt`:**
- `@Composable fun SearchScreen(onNavigateToNodeEditor: (String) -> Unit, onNavigateBack: () -> Unit)`
- Get ViewModel via `hiltViewModel<SearchViewModel>()`
- Collect state
- Scaffold with TopAppBar containing:
  - Back arrow button (onNavigateBack)
  - OutlinedTextField for search query (fills remaining width). Autofocus on screen entry. Hint text: "Search nodes..."
- Body based on state:
  - `Idle`: centered hint text "Type to search" or similar subtle instruction
  - `Loading`: centered CircularProgressIndicator
  - `Success`: LazyColumn of search result items. Each item shows:
    - Node content (first line, truncated to ~100 chars)
    - Note preview (if non-empty, smaller text, truncated)
    - Document title as breadcrumb label (small text, secondary color)
    - On click: `viewModel.onResultTapped(item)`
  - `NoResults`: centered text "No results for '{query}'"
  - `Error`: centered error text with retry option
- Collect side effects: `NavigateToNodeEditor` -> `onNavigateToNodeEditor(docId)`
