**3. Create ExportRepository interface** (`repository/ExportRepository.kt`):
```kotlin
interface ExportRepository {
    suspend fun exportAll(): Result<File>
}
```
Returns a `Result<File>` where the File points to the downloaded ZIP in cacheDir.

**4. Create ExportRepositoryImpl** (`repository/impl/ExportRepositoryImpl.kt`):
- Inject `HttpClient`, `@Named("baseUrl") baseUrl: String`, and `@ApplicationContext context: Context`
- `exportAll()`: use `runCatching` to GET `$baseUrl/api/export`, read response as `ByteArray` via `response.body<ByteArray>()`, write to `File(context.cacheDir, "outlinergod-export-${System.currentTimeMillis()}.zip")`, return the File.
- Follow the same pattern as SyncRepositoryImpl for Ktor usage.

**5. Register in RepositoryModule.kt**: `@Provides @Singleton fun provideExportRepository(impl: ExportRepositoryImpl): ExportRepository = impl`

**6. Update SettingsViewModel.kt** — add export functionality using the EXISTING `SettingsSideEffect` sealed class:
- Add `ExportRepository` to constructor injection (alongside existing `SettingsDao`, `AuthRepository`, `HlcClock`)
- Add `isExporting: Boolean = false` to SettingsUiState.Success data class (new field, default false)
- Add a new variant to the EXISTING `SettingsSideEffect` sealed class: `data class ShareFile(val filePath: String) : SettingsSideEffect()`. Do NOT create a separate `ExportSideEffect` sealed class — the container type remains `ContainerHost<SettingsUiState, SettingsSideEffect>`.
- Reuse the existing `SettingsSideEffect.ShowError(message: String)` for export failures (it already exists in the sealed class).
- Add `fun exportAllData() = intent {}`:
  - `reduce { (state as? SettingsUiState.Success)?.copy(isExporting = true) ?: state }`
  - Call `exportRepository.exportAll()`
  - On success: `val file = result.getOrThrow()` then `postSideEffect(SettingsSideEffect.ShareFile(filePath = file.absolutePath))`
  - On failure: `postSideEffect(SettingsSideEffect.ShowError(e.message ?: "Export failed"))`
  - Finally: `reduce { (state as? SettingsUiState.Success)?.copy(isExporting = false) ?: state }`

**Exact type flow chain for export:**
1. `ExportRepositoryImpl.exportAll()` returns `Result<File>` (File on disk in cacheDir)
2. SettingsViewModel on success: gets `file` from Result, posts `SettingsSideEffect.ShareFile(filePath = file.absolutePath)` (String path)
3. SettingsScreen handles the side effect: constructs `File(filePath)`, calls `FileProvider.getUriForFile(context, "${context.packageName}.fileprovider", File(filePath))` to get a content:// URI, then creates `Intent.ACTION_SEND` with that URI

**7. Update SettingsScreen.kt**:
- Add "Export All Data" button (OutlinedButton or similar) in the settings list, below the logout button
- Show CircularProgressIndicator when `(state as? SettingsUiState.Success)?.isExporting == true`
- Collect side effects: on `SettingsSideEffect.ShareFile(filePath)`:
  1. Construct file: `val file = File(filePath)`
  2. Get content URI: `val uri = FileProvider.getUriForFile(context, "${context.packageName}.fileprovider", file)`
  3. Create share intent: `Intent(Intent.ACTION_SEND).apply { type = "application/zip"; putExtra(Intent.EXTRA_STREAM, uri); addFlags(Intent.FLAG_GRANT_READ_URI_PERMISSION) }`
  4. Launch: `context.startActivity(Intent.createChooser(intent, "Export Data"))`
- On `SettingsSideEffect.ShowError(message)`, show a Snackbar or Toast

Note: SettingsScreen needs access to `LocalContext.current` for FileProvider and startActivity. This is standard Compose practice.
