```xml
<provider
    android:name="androidx.core.content.FileProvider"

```

**3. Create ExportRepository interface** (`repository/ExportRepository.kt`):
```kotlin
interface ExportRepository {
    suspend fun exportAll(): Result<File>
}
```
Returns a File pointing to the downloaded ZIP in cacheDir.

**4. Create ExportRepositoryImpl** (`repository/impl/ExportRepositoryImpl.kt`):
- Inject `HttpClient`, `@Named("baseUrl") baseUrl: String`, and `@ApplicationContext context: Context`
- `exportAll()`: use `runCatching` to GET `$baseUrl/api/export`, read response as `ByteArray` via `response.body<ByteArray>()`, write to `File(context.cacheDir, "outlinergod-export-${System.currentTimeMillis()}.zip")`, return the File.
- Follow the same pattern as SyncRepositoryImpl for Ktor usage.

**5. Register in RepositoryModule.kt**: `@Provides @Singleton fun provideExportRepository(impl: ExportRepositoryImpl): ExportRepository = impl`

**6. Update SettingsViewModel.kt**:
- Add `ExportRepository` to constructor injection
- Add `ExportSideEffect` sealed class: `ShareFile(uri: Uri, mimeType: String)`, `ShowExportError(message: String)`
- Change the container's side effect type to support both navigation and export side effects. If SettingsViewModel currently uses a sealed class for side effects, add the new variants. If it uses `Nothing`, change to a `SettingsSideEffect` sealed class.
- Add `isExporting: Boolean = false` to the UiState
- Add `fun exportAllData() = intent {}`:
  - `reduce { state.copy(isExporting = true) }`
  - Call `exportRepository.exportAll()`
  - On success: get URI via `FileProvider.getUriForFile(context, "${context.packageName}.fileprovider", file)` — but since ViewModel shouldn't hold Context, pass the File via side effect and let the Screen create the URI. Better pattern: create a `ExportSideEffect.ShareFile(filePath: String)` side effect, and the Screen handles FileProvider + Intent.
  - On failure: `postSideEffect(SettingsSideEffect.ShowExportError(e.message))`
  - `reduce { state.copy(isExporting = false) }`

**7. Update SettingsScreen.kt**:
- Add "Export All Data" button (OutlinedButton or similar) in the settings list, below the logout button
- Show CircularProgressIndicator when isExporting is true
- Collect side effects: on `ShareFile`, create Intent.ACTION_SEND with `type = "application/zip"`, add `FLAG_GRANT_READ_URI_PERMISSION`, use `FileProvider.getUriForFile()` to get the content URI from the file path, set as `EXTRA_STREAM`, and launch via `context.startActivity(Intent.createChooser(intent, "Export Data"))`
- On `ShowExportError`, show a Snackbar or Toast

Note: SettingsScreen needs access to LocalContext.current for FileProvider and startActivity. This is standard Compose practice.
