**SearchQueryParser** (`search/SearchQueryParser.kt`):
Create a data class `ParsedQuery(val ftsTerms: String, val isCompleted: Boolean?, val color: Int?, val inNote: Boolean, val inTitle: Boolean)`. Create `SearchQueryParser` object with `fun parse(rawQuery: String): ParsedQuery` that:
- Extracts `is:completed` / `is:not-completed` -> sets isCompleted Boolean
- Extracts `color:red` (1), `color:orange` (2), `color:yellow` (3), `color:green` (4), `color:blue` (5), `color:purple` (6) -> sets color Int
- Extracts `in:note` / `in:title` -> sets corresponding booleans
- Remaining text after extracting operators = ftsTerms (trimmed)
- Append `*` to the last word in ftsTerms for prefix matching (only if ftsTerms is non-empty and the last character is not already `*`)

**SearchRepository interface** (`repository/SearchRepository.kt`):
```kotlin
interface SearchRepository {
    suspend fun searchNodes(query: String, userId: String): List<NodeEntity>
}
```

**SearchRepositoryImpl** (`repository/impl/SearchRepositoryImpl.kt`):
- Inject `NodeDao` via constructor (`@Inject constructor(private val nodeDao: NodeDao)`)
- In `searchNodes()`: call `SearchQueryParser.parse(query)`, build SQL string:
  - If `inNote` only: `SELECT n.* FROM nodes n JOIN nodes_fts fts ON n._rowid = fts.rowid WHERE nodes_fts MATCH ? AND n.user_id = ? AND n.deleted_at IS NULL` with MATCH targeting `note:` prefix
  - If `inTitle` only: MATCH targeting `content:` prefix
  - Otherwise: standard MATCH on both content and note
  - Append `AND n.completed = {0|1}` if isCompleted is non-null
  - Append `AND n.color = {colorInt}` if color is non-null
  - `ORDER BY bm25(nodes_fts)`
- Use `SimpleSQLiteQuery(sql, arrayOf(ftsQuery, userId))` — parameterized to prevent injection
- Return `nodeDao.searchFts(query)`
